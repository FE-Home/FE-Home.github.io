# 符合 eslint、sasslint 的自动格式化工具

<blockquote class="blockquote-center">邓浩博</blockquote>

> **为什么要使用 Prettier 工具？**

> 在多人项目中，大家每个人都有自己的代码习惯：运算符前后空格、大括号位置、行尾分号、缩进规则、代码块间空行等。。。风格的不一致，会导致一个项目中不同人接收过的模块有不同的代码格式，导致阅读、维护的困难，同时在提交代码的时候还很容易出现冲突。

> 之前我们使用 lint 类工具对代码风格做强定义，这样子会极大减少大家输出不一致的风格代码。但是这种强制大家一致书写习惯的方式，也会导致长期习惯了另外一种代码风格的人，开发效率的降低。他会把很多注意力，放在解决 lint 工具的警告、敲击空格键上。

> prettier 支持众多编辑器，其中本人重点推荐 vsCode（不要问我为什么）

### **干货：**

1. 广告：vsCode + prettier 专治代码洁癖

2. vsCode 配置：vsCode 中安装 prettier、eslint、sasslint、stylelint 插件，并按照插件的使用要求去安装对应的东西

3. vscode 中 setting 配置：

```javascript
"editor.formatOnSave": true,`
`"files.autoSave": "afterDelay",
"prettier.eslintIntegration": true, //让prettier使用eslint的代码格式进行校验
"prettier.stylelintIntegration": true, //让prettier使用stylelint的代码格式进行校验
"prettier.semi": false, //去掉代码结尾的分号
"prettier.singleQuote": true, //使用带引号替代双引号
"eslint.autoFixOnSave": true,
"eslint.options": {
  "extensions": [".js", ".vue"]
},
"eslint.validate": [
  "javascript",
  "javascriptreact",
  {
    "language": "html",
    "autoFix": true
  },
  {
    "language": "vue",
    "autoFix": true
  }
]
```

4. 项目配置：目录下需要增加.eslintrc .stylelintrc .sasslintrc, prettier 会按照.eslintrc 和.stylelintrc 的规则去自动格式化代码，那么.sasslintrc 是干什么用的呢，这是金融线上 fdd 项目的配置，暂时是挪用了，在这里是用来再次校验一下通过 stylelint 格式化后的代码是否符合 sasslint 规范，仅此而已，不过理论上来讲 stylelint 中可以安装 sass 的插件，所以通过 stylelint 就可以配置 sass 的规范了，应该完全可以用 stylelint 替代 sasslint，_下面附上我的项目的配置_：
